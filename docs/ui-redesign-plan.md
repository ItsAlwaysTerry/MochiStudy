## 一、审查结论：为什么现在显得"原始粗糙"

功能没问题，粗糙感来自五个可定位的根因：

### 根因 1：设计 token 形同虚设（最大根因）

`style.css` 顶部定义了一套完整 token，但全文件 11438 行里：

- `--radius-card` 只被引用 **2 次**，硬编码圆角却有 **12 种**（999/12/10/8/14/16/18/20/6/3/5px…）
- 硬编码 hex 色值 **100+ 处**（#f5c518 金、#4a4580 紫、#e07020 橙、#9e9e9e 灰…各自为政）
- 字号 **30 种**
- JS 模板里内联 `style="…"` **113 处**（app.js 77 处最多）

→ 每张卡片像不同人画的。这是"高级感"缺失的第一来源：高级感 = 重复与克制。

### 根因 2：强调色互相打架，没有主次

首页同屏出现：紫（任务标题/物理tab）、梅红（主按钮）、粉（chip/导航）、橙金渐变（能量浮窗）、绿（农场文字）、米黄（callout）。用户视线没有"最重要的一个颜色"可跟随。

### 根因 3：盒子套盒子，层级全靠边框和阴影

任务卡里套队列项卡、再套步骤 chip 条、再套米黄提示块、再套"学习材料与记录"行——**4~5 层嵌套框**，每层都有自己的边框/底色/圆角。现代 APP 用留白和字号分层，不用框套框。

### 根因 4：浮层无纪律

- 「今日能量」浮窗默认展开，直接压住首页导入框和「开始专注」主按钮（`01-home-desktop-top.png` 铁证）
- toast 固定右下，压设置页按钮、压勋章列表（`05/06` 多张）
- 抽奖是深色赌场风，与浅色主站完全两个世界（`11`）

### 根因 5：同形状 ≠ 同功能，操作语义混乱

学习档案页顶部**三行药丸**：第一行是页面级 tab（今日/复习/档案）、第二行是科目筛选、第三行是动作按钮（从零重学/啃卷子/出测验/导出）——形状全是圆角药丸，但填充色一个梅红、一个紫、一个粉、一个深梅，用户无法从形状判断"这是切换还是执行"。

### 各页具体病灶清单（codex 初审 + Claude 复核）

| 页面 | 病灶 | 证据 |
|---|---|---|
| 首页 | 能量浮窗压住导入框/开始专注；一屏 6+ 个目标抢注意力；农场地块深棕色块发闷 | 01-top/bottom |
| 首页移动 | 卡片嵌套在 390px 下压迫感强；步骤条贴底部导航 | 12 |
| 复习 tab | 行内"开始"按钮权重过大；移动端复习原因被截断成省略号（原因是决策依据，不能截） | 02、13 |
| 学习档案 | 三行药丸语义混乱；核心摘要 4 列 grid 局促；「测这个知识点」整宽深色巨钮权重失衡 | 03 |
| 今日 tab | 下方留白过大；统计与内容关系弱 | 04 |
| 勋章页 | x0 勋章行是灰底大条，看着像禁用按钮/死内容；抽奖机会横幅占首屏、真勋章下沉；toast 压列表 | 05、14 |
| 设置页 | 超长、密度不均；滚动时 sticky header 截断内容；表单控件风格不一 | 06 全部 |
| 专注 overlay | 「放弃本轮」和导入入口对比度过低接近不可见 | 08 |
| deciding 屏 | 按钮换行两行、重心散 | 09 |
| 抽奖 | 深色赌场风与主站割裂（保留玩法，风格需搭桥） | 11 |

## 二、设计方向（Claude 推荐，可否决）

**「暖白极简」**：保留奶油暖底的品牌性格（贴合收集/农场/温和激励的用户画像），但把噪声全部拆掉——

1. **一个主色**：梅红 `#864d61` 是唯一行动色（按钮/激活态/链接）。紫色降级为物理科目的识别色，金橙只准出现在"奖励语境"（能量/抽奖/勋章图标），绿只做成功语义。
2. **科目色只做微识别**：小圆点、2px 左边框、细描边 pill；不准再刷大面积。
3. **卡片减负**：页面级卡 = 白底 + 大圆角 + 极轻阴影（无边框）；卡内子块 = 浅米底色块（无阴影无边框）；**最多两层面**，第三层起只用留白+字号分层。
4. **呼吸感**：区块间距拉大（24/32px），卡内 padding 统一 16/20px，行高 1.5+。
5. **克制的动效**：统一 160ms ease-out；hover 只变阴影/底色，不乱位移。

## 三、Token 与组件规范（Phase 0 落地物）

### 3.1 新 token（替换/收编现有 :root）

```css
/* 色彩角色 */
--bg: #fdf9f2;            /* 略提亮的奶油底 */
--surface: #ffffff;
--surface-dim: #f7f1e8;   /* 卡内子块唯一底色 */
--ink: #2b2226;           /* 主文字 */
--ink-2: #6b5a60;         /* 次要文字 */
--ink-3: #a39298;         /* 弱化文字/占位 */
--line: rgba(43,34,38,.08);   /* 唯一分隔线色 */
--primary: #864d61;  --primary-ink: #fff;
--primary-tint: #f6e8ed;      /* 主色浅底（激活态/hover） */
--gold: #d9a13f;              /* 仅奖励语境 */
--good: #3f9d6b;  --warn: #e0a93a;  --bad: #c96b6b;
--math: #c2703f;  --physics: #4a4580;  --chem: #2f6a3f;  /* 仅微识别 */

/* 圆角：只留 3 档 */
--r-lg: 20px;   /* 页面级卡 */
--r-md: 12px;   /* 子块/输入框/按钮 */
--r-full: 999px;/* chip/pill */

/* 阴影：只留 2 档 */
--shadow-1: 0 1px 3px rgba(43,34,38,.06), 0 4px 16px rgba(134,77,97,.06);
--shadow-2: 0 8px 32px rgba(43,34,38,.16);  /* 仅浮层 */

/* 间距：4 的倍数 */
--sp-1:4px; --sp-2:8px; --sp-3:12px; --sp-4:16px; --sp-5:24px; --sp-6:32px;

/* 字阶：只留 7 档 */
--fs-xs:12px; --fs-sm:13px; --fs-base:14px; --fs-md:16px;
--fs-lg:20px; --fs-xl:26px; --fs-num:32px; /* 数字展示 */
```

### 3.2 组件规范（形状 = 功能）

| 组件 | 规范 | 取代现状 |
|---|---|---|
| `.btn-primary` | 梅红填充，40px 高，--r-md | 各处深色/粉色大按钮 |
| `.btn-tonal` | --primary-tint 底 + 梅红字 | 各处 soft 按钮 |
| `.btn-ghost` | 透明底 + --line 描边 | 各处 outline 按钮 |
| `.btn-sm` | 32px 高变体 | 行内小按钮 |
| `.seg` 分段控制器 | 一个浅槽内滑块式切换，用于**页面级 tab 和科目切换** | 学习页/档案页两行药丸 |
| `.chip` | --r-full 小药丸，仅表**筛选/标签/状态** | 混用的各种 pill |
| `.card` | 白底 --r-lg --shadow-1 无边框 | 边框+阴影混用的卡 |
| `.card-sub` | --surface-dim 底 --r-md 无阴影 | 卡内嵌套小卡 |
| `.field` | 统一输入框/textarea/select 样式 | 三种表单风格 |
| toast | **顶部居中**下落，2.4s 自动走 | 右下角压内容 |

规则：**同一页面内，切换用 seg、执行用 btn、状态用 chip，不允许混形**。

## 四、分阶段执行计划（每阶段 = 一个 codex 任务包）

> 每阶段固定流程：codex 改 → `node --check` 全部 JS → codex 用 Chrome/CDP 截同角度前后对比图 → Claude 看 git diff + 截图验收 → bump `?v=` 缓存版本号 → 用户肉眼确认 → 才进下一阶段。

### Phase 0 — 地基：token + 组件库 + 机械迁移（无观感巨变，纯还债）

1. 在 style.css 顶部落地 3.1 token 和 3.2 组件类（新增，不删旧类）。
2. 机械替换：12 种硬编码圆角 → 3 档 token；30 种字号 → 7 档；100+ hex → 色彩角色 token（映射表 codex 生成、Claude 抽查）。
3. JS 模板 113 处内联 `style="…"` 迁入 CSS 类（`app.js` 77 处优先）。
4. 验收标准：截图与改前逐张对比，允许"更统一"的微差，不允许布局破坏。

### Phase 1 — 全局秩序：导航/浮层纪律（观感提升最大的一步）

1. **能量浮窗默认收起**为小胶囊（图标+可抽奖数），展开态限制最大高度且**永不覆盖**导入框/主按钮（改默认停靠位 + 拖动边界）；有新奖励时胶囊轻微脉冲提示。
2. toast 移到顶部居中。
3. 顶栏去掉"卡片浮起"感（融入背景，只留品牌字 + 设置钮；build 号移入设置页关于）。
4. 侧边栏/底部导航激活态统一为 `--primary-tint` 底 + 梅红字。
5. z-index 清册：写一段注释登记所有浮层层级，杜绝互压。

### Phase 2 — 首页：一屏一件事

1. 信息层级重排：**暑假任务卡 = 唯一主角**（保持现有内容，减嵌套：队列项从"卡中卡"改为分隔线列表，步骤 chip 融进行内）；streak 横幅压成一行轻量条。
2. 右栏合并为一张「成长」卡：迷你农场 + 本周趋势合体（农场地块底色提亮，摆脱深棕闷块）。
3. 导入框降级为轻量子块（暑假任务流内已有导入口，首页这个是兜底），但**不删除**。
4. 「开始专注」成为右栏最显著按钮。
5. 移动端：嵌套减一层后自然缓解；另修步骤条贴底部导航的间距。

### Phase 3 — 学习页三 tab

1. 顶部改 `.seg` 分段控制器（今日/复习/档案 一体成型），科目切换也用 seg，动作按钮统一 `.btn-ghost` + 图标收进一行；三行药丸消失。
2. 复习列表：行内「开始」改 `.btn-sm`；移动端复习原因**不许截断**（换行显示，按钮上移到行首右侧）。
3. 档案：核心摘要 4 列 grid → 上下两行宽松布局；「测这个知识点」降为 `.btn-tonal` 常规宽度；卡片详情嵌套减一层。
4. 今日 tab：统计条紧凑化，留白回收。
5. 红线：**不删除任何按钮/入口**（从零重学、啃卷子、出测验、导出、多选测验都保留）。

### Phase 4 — 勋章页 + 设置页

1. 勋章：抽奖机会横幅压成一行（数字+去抽奖按钮）；x0 勋章行改"轻空态"（白底 + 灰字 + 图标去饱和），有进度的行才用实底；列表上移进首屏。
2. 设置：分组收进 `<details>` 手风琴（阅读外观/提醒音/AI 指南/数据/更新/关于六组，默认全收起），页顶加锚点行；统一 `.field` 表单样式；修 sticky header 截断。

### Phase 5 — 沉浸态 + 抽奖 + 移动端终扫

1. 专注 overlay：「放弃本轮」提到可见对比度（ghost 白描边）；deciding 按钮改纵向排列不换行；导入入口给个明确的半透明卡底。
2. 抽奖：保留深色仪式感（这是特意的"特殊场合"），但配色桥接品牌——底色从黑紫调向深梅 `#2d1420`，金色统一用 `--gold`，按钮/文字复用主站组件形状。
3. 移动端全页面过一遍：间距、截断、点击目标 ≥40px。
4. 全站最终一致性 sweep + 死 CSS 清理。

## 五、执行铁律（写给 codex 的每个任务包都要带）

1. **只动样式和模板结构，不动数据层**：study_log/备份格式/localStorage key 一个都不许碰。
2. **不删除任何功能入口**（历史教训：复制图片按钮事故）。精简 ≠ 删功能，拿不准的先问。
3. 精灵图保持 `image-rendering: pixelated` + `mix-blend-mode: screen`，不许换 SVG/emoji。
4. **禁止用 PowerShell 管道处理含中文的文件**（会 UTF-8 乱码，本次截图任务已再次踩过）；写文件用 UTF-8 脚本或 apply_patch。
5. 每轮改完：`node --check` 全部 JS + bump `index.html` 的 `?v=` 版本号。
6. 每阶段产出前后对比截图存 `docs/review-batch/screenshots/ui-phase-N/`，Claude 验收后才算完成。
7. 游戏数值/文案一概不改，本次只做视觉与交互结构。
