# 暑假物理执行计划（主线视频 + 卡住救急 + 例题留痕）

> 状态：工作版，后续持续迭代。
> 日期：2026-07-14
> 目标学生：物理约 20-30 分，概念、公式、读题、受力分析都很薄弱；自主规划能力弱，适合被明确安排。

---

## 1. 当前判断

学生现在最大的问题不是缺资源，而是缺一个低自主性也能执行的闭环：

```text
看主线视频 -> 做过关题检验 -> 卡住才翻实体书/基础课 -> 导入 MOCHI-RECORD -> 后续复习
```

不能把“黄夫人基础课/一轮课”当主线从头刷。它足够基础、足够细，但总时长过长，学生会被淹没。
也不能只跟小红书推荐视频走，因为这些视频没有统一讲义和对应练习，容易“看完了但没学会”。

所以资源分工定为：

| 资源 | 定位 | 使用方式 |
|---|---|---|
| 小红书攻略推荐视频 | 主线 | 每天只看指定视频/指定片段 |
| 一轮复习讲义 | 卡住救急 + 课后练题 | 听课或做题卡住时翻对应页；不会就看答案、抄答案、说一句人话解释 |
| 高一高二基础讲义 | 第二层救急 | 一轮页还是不懂时，再看对应基础概念页 |
| 黄夫人基础视频 | 最后救急 | 只看指定概念视频，不从合集开头刷 |
| MochiStudy | 任务、打卡、可视化、复习入口 | 完成必须导入 MOCHI-RECORD |

## 2. 每天物理标准流程

单日物理建议 90-120 分钟；状态差时压缩到 60 分钟，但不能只看视频。

### 标准版

1. **主线视频，25-45 分钟**
   - 跟小红书攻略推荐视频走。
   - 只看当天指定视频或片段，不自由扩展。
   - 如果听得下去，不要求先翻实体书。

2. **视频例题留痕，5 分钟**
   - 只截 1-3 张“这节课最代表性”的例题/方法截图。
   - 不要求截全，不要求做笔记长文。
   - 正式目标：以后放进网站的每个视频任务下面。

3. **过关练习，25-40 分钟**
   - 优先做网站里的过关小题。
   - 能独立说出思路并做对，说明这节真正过关。
   - 如果只是听懂但不会做，先补同类题，不急着补长课。
   - 题目做不会也要记录卡点，不允许只看答案结束。

4. **卡住救急，10-30 分钟**
   - 听课时概念跟不上，或做题时某个概念说不清，再翻实体书指定页。
   - 一轮讲义仍看不懂，再看基础讲义或黄夫人基础视频的指定条目。
   - 到时间仍不懂，记录卡点，先不要把一天拖死。

5. **导入记录，5 分钟**
   - 用 MOCHI-RECORD 记录：今天学了什么、卡在哪里、几星。
   - 没有 MOCHI-RECORD，不算完成任务。

### 压缩版

```text
主线视频 25 分钟 -> 过关题 20 分钟 -> 卡住才翻书 10 分钟 -> 导入记录 5 分钟
```

## 3. 例题截图留痕方案

### 正式目标：做进网站

每个暑假视频任务下面增加一个“我在视频里学到的例题”区域。

学生只做三件事：

1. 截图。
2. 粘贴到当前视频任务下面。
3. 标记：`会 / 半会 / 不会`。

系统记录：

- 对应视频任务
- 对应知识点
- 截图
- 学生状态：会 / 半会 / 不会
- 是否已经让 AI 出过同类题
- 是否已经导入 MOCHI-RECORD

后续复习时，可以直接筛：

```text
拿出所有“平抛运动”里半会/不会的例题
拿出所有“电路”里还没出过同类题的例题
```

实现注意：

- 不放进 `study_log`，避免破坏已有学习记录结构。
- 图片不要塞 localStorage；正式实现应考虑 IndexedDB。
- `study_log` 仍只保存最终学习记录。

### 临时方案

网站功能没完成前，继续用：

```text
docs/summer-physics-examples/例题收件箱.md
```

但这个文件更适合家长/Codex 整理，不适合作为学生长期操作入口。学生文件操作弱，正式还是应进网站。

## 4. 卡住救急规则

2026-07-14 学生反馈：前两天主线视频基本能听懂，讲得细；偶尔不懂会看弹幕获得更通俗解释。这个反馈说明实体书不应该作为硬性课前任务，否则会增加负担。

所以现在改成：先跟主线视频和过关题走；只有听课或做题卡住时，才按这个顺序补基础：

```text
一轮讲义对应页
  -> 仍不懂：高一高二基础讲义对应页
  -> 仍不懂：黄夫人基础视频指定条目
  -> 到时间仍不懂：先进入主线视频，卡点导入 MOCHI-RECORD
```

关键原则：

- 不追求把黄夫人全部看完。
- 不允许因为“概念还没补完”无限拖延主线。
- 救急补基础最多 30 分钟。
- 每次只补当天要用的概念。

学生要输出的不是漂亮笔记，而是低摩擦的一句话：

```text
加速度：速度变化得快不快，不是速度大小。
合外力：所有力合起来之后真正让物体加速的那个效果。
电动势：电源把电荷往外推的本事，不等于电压表随便读到的电压。
```

## 7. 网站迭代方向

下一步不是做大资源库，而是把每个暑假任务卡变成一个可执行学习单：

```text
卡住再看
  - 今天可能卡住的 3-5 个概念
  - 实体书救急页码
  - 基础讲义/基础视频备用入口

  - 视频链接
  - 建议看多久

例题留痕
  - 截图粘贴区
  - 会 / 半会 / 不会

过关练习
  - 网站过关小题
  - 一轮讲义指定题

完成
  - 导入 MOCHI-RECORD
```

实现顺序建议：

1. 先把现有 7 节任务加上“卡住概念/实体书救急页码/备用视频”字段。
2. 再做“例题截图粘贴区”的最小版。
3. 最后把截图例题接到复习和 AI 出同类题。
