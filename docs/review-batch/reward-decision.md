# #5 奖励系统：去赌性 + 统一（决策文档，等你拍板）

> 过夜没动这块代码——奖励涉及价值观判断，必须你先选方向。下面三个方案，选一个（或组合），我再落地。

## 现状（问题在哪）

1. **两套奖励并存**
   - 主抽奖：`lottery_config` / `achievement_state`（勋章 → 抽奖次数 → 转盘）。
   - 暑假能量：`summer_reward_config` + `summerTasks.js` 里的骰子棋盘。
   - 学生要理解两套完全不同的奖励循环，认知重复。

2. **暑假奖励是"真金 + 概率 + 差点中大奖"**
   - 奖池：`SUMMER_REWARD_PRIZES` = 再接再厉(0元,权重40) / 5元(30) / 10元(20) / 20元(7) / 50元(2) / 100元(0.8) / 200元(0.2)。
   - 玩法：8 格棋盘、骰子滚 ~4.5 秒、**near-miss（差点中大奖）**、中大奖金光庆祝。
   - 对一个"缺自信"的孩子，这是**赌博框架**：动机从"我学会了"被偷换成"赌一把"。near-miss 在老虎机里就是留人设计，用在孩子身上，"就差一点没中 200" 更可能是挫败。

3. **进度能被"没学也刷出来"**
   - `taskStepScore()`（summerTasks.js ~914 行）把 `exampleQuizPromptCopiedAt` / `promptCopiedAt`（复制过测验包）也算进度。
   - 结果：抽奖机会能靠"点复制"刷出来，不需要真学。

---

## 方案 A：保守（保留骰子玩法，只去毒）

**改什么**
- 删掉 near-miss 差点中大奖的动画/逻辑（`showBigWinCelebration` 前的 near-miss 段）。
- 抽奖机会只认**真实导入的 MOCHI-RECORD**，不认 `promptCopiedAt`（改 `availableRewardClaims` 的判定基于 `routeDayCompleted` 已经是真完成，但 `taskStepScore` 的进度环要改成只认 examples+import+reflection，不认复制）。
- 暑假期把主抽奖入口隐藏，避免两套并存。

**优点**：改动最小，保留了"摇骰子"的趣味。
**缺点**：真金 + 概率的赌博内核还在，只是没那么露骨。对缺自信画像仍有风险。

---

## 方案 B：去赌性（推荐）—— 达成即得的固定奖励

**核心**：把"随机抽钱"换成"**里程碑确定奖励**"。
- 完成一天 / 一周 → **固定**奖励（家长和孩子事先约定，金额写在 `summer_reward_config`，但**非随机**）。
- 例：每完成 1 周 = 家长约定的固定奖励（钱 / 一次出去玩 / 一个小物件，家长自己填）。
- **删掉**：骰子、8 格棋盘、概率权重、near-miss。
- **保留**：达成时的庆祝动画（金光/撒花）——庆祝"你做到了"，不是"你赌中了"。
- 结果卡照样能截图发家长（"本周达成，兑现约定奖励"）。

**为什么最适合这个用户**
- 缺自信的孩子需要"努力→确定回报"的可预期性，赌博的间歇性强化会让他更焦虑。
- "收集/陈列"性格：把达成做成可陈列的徽章/兑换券，比抽奖更契合。
- 家长友好：家长知道孩子做到什么就该给什么，不用解释概率。

**改什么（落地时）**
- `SUMMER_REWARD_PRIZES` 概率池 → 改成里程碑表（day/week → 固定奖励文案，金额可空、可自定义）。
- `startSummerRewardDraw` / `beginSummerRewardRoll` / `finishSummerRewardDraw` 骰子流程 → 换成"达成即展示固定奖励卡 + 庆祝动画 + 写入 history"。
- `taskStepScore` 进度只认真实痕迹。
- `renderSummerRewardHistory`（app.js ~4024）文案同步。

**缺点**：少了"抽奖惊喜感"。可用"连续达成解锁更大约定奖励"补偿仪式感。

## 方案 C：彻底简化 —— 只留一套奖励

**核心**：删掉暑假独立奖励，暑假的完成度直接喂给**现有主勋章/抽奖系统**。
- 完成暑假节点 → 触发 `checkAndGrantAchievements()`（现成机制），累计勋章 → 主转盘抽奖。
- 全站只有一套奖励循环，认知负担最低。

**优点**：最少的系统、最一致。
**缺点**：主抽奖也是概率玩法（同样有赌性问题，只是不是真金）；且暑假的"专属激励感"被稀释。

## 我的推荐

**B（里程碑确定奖励）** 为主，去掉骰子和 near-miss；如果你舍不得"摇一摇"的趣味，退而求其次选 **A**（至少先把 near-miss 和"复制即刷进度"两个毒点去掉）。**C** 只在你想极简、且能接受主抽奖那套概率玩法时选。

**无论哪个方案，两件事都建议无条件做**：
1. 删 near-miss（差点中大奖）。
2. 进度/机会只认真实导入记录，不认"点了复制"。
