# 统一奖励经济系统设计（v1，待 codex 模拟调参）

> 用户授权全权设计，给了预算与思想；我自审 2 轮后给方案，再派 codex 验预算/实测，我审计。
> 预算硬约束：**≤¥40/天**、**≤¥150/阶段（约1周）**。思想：高频小正反馈 + 偶发大奖梦 + 不花太多钱 + 不让人老不中大奖。

## 一、当前问题（为什么要重做）

1. **两套发钱系统并存、预算失控**：主抽奖（勋章→券→¥20–200）+ 暑假骰子奖励（每科独立、真金），加起来没有任何预算上限。
2. **导入膨胀**：以前 1 题=1 条记录，现在 1 次导入 = 4–6 条（Gemini 每知识点出 1–2 题确认）。所以"每 10 条记录=1 小勋章、每 50 条=1 大勋章"等阈值**快了约 5 倍**，勋章和抽奖券发太多。
3. **暑假奖励每科独立**：数/物/化 各存各的 reward，学生要分三份看进度，且颗粒度粗（只有"整天/整周全完成"才触发）。

## 二、目标架构：一套经济、两个发钱渠道、跨科统一

```
今日能量（跨 数/物/化 汇总）
 ├─ 渠道A 每日小奖（确定性，高频小额，日预算池 ≤40/天）
 └─ 渠道B 抽奖（唯一一套抽奖=大奖梦，券来自 勋章 + 阶段里程碑，阶段预算池 ≤150/阶段）
勋章系统 = 荣誉/收集层（重新校准阈值 + 新增暑假勋章）→ 产出抽奖券喂给渠道B
```

- **删掉**暑假每科独立骰子奖励；**保留并统一**成一套抽奖（就是现有主抽奖，改奖池+加预算上限）。
- 能量浮窗改为**跨科统一**：显示今日能量、今日已得小奖、阶段进度、抽奖券数 + 去抽奖。

## 三、今日能量（统一度量，抗刷）

- **今日能量 = 当天完成的任务节点数（跨三科合计）**。一个"节点"= 一个视频走完整流程（截图+测验+本节收尾=真功夫，难刷）。
- **不用原始记录条数计能量**——正好绕开"导入膨胀"问题（记录条数会灌水，节点数不会）。
- "达标日" = 当天能量 ≥ 2 个节点。

## 四、渠道A：每日小奖（确定性，日预算 ≤40）

当天按能量发一次（取最高档；当天后面又多做，补差价）：

| 今日节点数 | 小奖 |
|---|---|
| 1 | ¥0（只给鼓励语「今天开张了，继续」，不发钱） |
| 2 | ¥5 |
| 3 | ¥8 |
| 4+ | ¥12（当日此项封顶） |

- **每日券 ×1**：当天能量 ≥ 2 节点即得 1 张「日常券」。
- 日常券抽奖池（小额）：`¥2(w35) / ¥5(w35) / ¥10(w22) / ¥20(w8)`；期望≈¥6.3/次，单次封顶¥20。
- 渠道A 每 study-day 期望 ≈ ¥8（小奖）+ ¥6（券）= **¥14/天**，硬上限 ¥40 兜底。

## 五、渠道B：抽奖（大奖梦，阶段预算 ≤150）

- **阶段 = 累计 5 个达标日**（不必连续；契合"松散但在学"的真实节奏）。
- 阶段达成 → 固定 **¥25** + **阶段大券 ×1**。
- 阶段大券抽奖池（大额）：`¥20(w50) / ¥50(w35) / ¥100(w15)`；期望≈¥42.5/次，单次封顶¥100。
- **连续加成**：连续 3 个达标日 → 额外「日常券 ×1」（鼓励连续性，小激励）。
- 渠道B 每阶段期望 ≈ ¥25 + ¥42.5 = **¥67.5/阶段**。

## 六、预算核对（第 2 轮自审重点）

假设：弱基础学生暑假 ~5 study-day/周、~2–3 节点/天 → 约 1 阶段/周。
- 渠道A：5 × ¥14 = **¥70/周**
- 渠道B：**¥67.5/阶段** ≈ ¥67.5/周
- **合计期望 ≈ ¥137/周** < 150 ✅（但偏紧，靠 codex 模拟确认尾部分布不会常爆）

**两个预算池分开管（关键，避免阶段达成日爆单日40）：**
- **日池 ≤¥40/天**：只管渠道A（今日小奖 + 日常券抽奖）。
- **阶段池 ≤¥150/阶段**：管渠道B（阶段固定奖 + 阶段大券），大券抽奖**不受日40限制**（大奖本就是里程碑，不该被日限卡死），只受阶段150限制。
- 每次发钱前查对应池：`已发+本次 > 上限` 则本次封顶到剩余额度；剩余为0则提示「本阶段奖励已达上限，下阶段继续」（极少触发，纯兜底防跑飞）。

**第 2 轮自审发现并已修正的坑：**
1. 原设计大券即时抽会和当日小奖叠加爆 ¥40 → 改为大券走阶段池、不受日限。
2. 原设计日常券期望偏高(¥10+) → 下调奖池到期望 ¥6.3，避免周总超 150。
3. ¥0「谢谢参与」会打击缺自信学生 → 日常券最低 ¥2（永远有正反馈），阶段券最低 ¥20。
4. 大奖 ¥100 概率定 15%（不是 3–5%）→ 满足"努力一个阶段就有实感机会中大奖、不老不中"。

## 七、勋章系统改造

### 7.1 重新校准（导入膨胀 → 阈值上调约 5×，只调"按记录条数"的项）
| 项 | 旧 | 新 |
|---|---|---|
| 小·recordCount（每N条记录） | 10 | **50** |
| 大·totalRecords（每N条） | 50 | **200** |
| 大·nodeRecords（单点每N条） | 20 | **40** |
- 不动 focusHours / studyDays / balancedWeeks / harvests（这些不随导入膨胀）。

### 7.2 新增暑假计划勋章（进现有勋章体系，荣誉+收集）
- 小勋章：每完成 **5 个任务节点**（跨科）→ 1 枚；每 **3 个连续达标日** → 1 枚。
- 大勋章：**完成任一科整条路线** → 1 枚；每完成 **1 个阶段（5 达标日）** → 1 枚。
- 数据源：统一 `summer_reward` 状态（节点数、达标日、阶段、路线完成）。

### 7.3 勋章→抽奖券（渠道B 的票来源）
- 勋章仍按"每 X 小勋章 / 每 1 大勋章 = 1 券"换券，但因 7.1 校准后勋章发放速率正常化，券速率也正常。
- 阶段里程碑另发阶段大券（见五）。**全部真金只从渠道A/B 出，勋章本身不直接发钱**——单一预算口，可控。

## 八、统一 3 科（数据层）

- 新增独立 key `summer_reward`（不进 `summer_task_state` 的 subject 分桶）：存能量、达标日、阶段进度、券、已发预算（日/阶段）、历史。
- 能量/达标日按**当天跨三科**的节点完成统计；`applySubjectContent` 不影响它（它遍历三科 state 桶汇总）。
- 迁移：旧各科 `state.reward` 只停用不删（不丢历史），新逻辑读写 `summer_reward`。

## 九、落地阶段（codex 分工）

- **P0 codex 模拟**：按上面数值跑蒙特卡洛（模拟 1000 个"周"，随机 study-day 数 3–6、节点数 1–4），输出：日均/周均/单日最大/单周最大 发放额，确认 ≤40/天、≤150/周的**尾部**也守得住；给出建议权重微调。→ 我据此定终值。
- **P1 我实现**：`summer_reward` 统一状态 + 能量度量 + 渠道A 每日小奖 + 预算池 + 券逻辑。
- **P2 我实现**：抽奖池改造（日常券池/阶段券池）+ 能量浮窗统一 UI。
- **P3 我实现**：勋章校准 + 新增暑假勋章。
- **P4 codex 浏览器实测** + 我审计。

## 十、待验证 / 风险
- 数值是 v1 提案，**必须以 codex 模拟结果为准**再定终值。
- 预算是"期望内 + 硬上限兜底"，不是每天精确 40；硬上限保证绝不跑飞。
- 需确认真实 study 频率假设（5天/周、2–3节点/天）；偏差大则整体缩放。

# 十一、最终定稿（第 3 轮自审 + 本地模拟验证，变体 C）

第 1 轮模拟发现两个真缺陷（假¥100大奖、缺周上限），据此改结构；本地跑 2 万周验证，锁定变体 C。

## 最终数值
- **预算上限**：每自然周硬上限 **¥150**、渠道A每日硬上限 **¥40**。
- **今日能量** = 当天跨三科完成的任务节点数；达标日 = 当天 ≥ 2 节点。
- **渠道A 今日小奖**（确定性）：2 节点→¥3，3→¥5，4+→¥8。
- **日常券**（能量≥2 得1张；连续3达标日额外+1张）抽奖池：`¥2 / ¥5 / ¥10 / ¥20`，权重 `45 / 35 / 15 / 5`（期望≈¥4.8）。
- **阶段** = 累计 5 个达标日 → 固定 **¥15** + 阶段大券 ×1。
- **阶段大券**抽奖池：`¥20 / ¥50 / ¥100`，权重 `52 / 33 / 15`（期望≈¥40.9，单次封顶¥100）。
- 大券抽奖不受日¥40限制，只受周¥150限制。

## 模拟验证（20000 周，固定种子）
| 指标 | 值 | 判定 |
|---|---|---|
| 周均实付 | ¥77.08 | ✅ 远低于150，约半预算，保守安全 |
| 周中位 / P95 / 最大 | ¥77 / ¥150 / ¥150 | ✅ 周¥150硬上限彻底兜住（原¥254已修） |
| 日均 / 渠道A日上限触发 | ¥17.5 / 0.05% | ✅ 日常几乎不被砍 |
| 周上限触发率 | 10.98% | ⚠️ 约1/10周顶到150（都是大奖周，正好卡在预算，不超） |
| ¥100 命中频率 | 每 6.6 阶段（≈9周）1 次 | ✅ 大奖梦真实、不老不中 |
| **¥100 全额发出率 / 平均实付** | **72.7% / ¥95.7** | ✅ 大奖是真的（原来0%全额、只发¥50，已修） |

## 体感节奏
- 每个 study-day：¥3–8 小奖 + 小额券抽（几乎天天有正反馈）。
- 约每 1.4 周：完成一个阶段，固定 ¥15 + 中大额抽奖。
- 约每 9 周：抽中 ¥100 大奖（暑假约能中一次）。
- 月均支出 ≈ ¥330（周77×4.3），在"周150"预算内、偏保守。

## 结论
数值锁定为变体 C。周硬上限保证绝不跑飞；日常精简、大奖真实且周期性可达，符合"高频小正反馈 + 罕见真大奖 + 不超预算"的全部要求。进入实现阶段。

# 十二、预算上限怎么写清楚 + 到顶后怎么继续激励（build 20260716e）

## 两条硬上限（钱绝不跑飞）
- **每日 ¥40**：渠道A（今日小奖 + 日常券抽奖）当天实付总额不超过 ¥40。阶段大券抽奖**不占**这条（大奖是里程碑，不该被日限卡死）。
- **每周 ¥150**：一整周（周一起算）所有渠道实付总额不超过 ¥150。**所有**发钱（含阶段大奖）都受这条约束。
- 代码里由 `syncEconomy()`/`econDraw()` 的 `pay()` 兜底：`已发 + 本次 > 上限` → 本次只发到剩余额度；剩余为 0 → 本次实发 ¥0。

## UI 里"写清楚"（学生/家长一眼看懂）
- 浮窗头部副行：`今日 N 个节点 · ¥X/40` —— 当天已得 / 当日上限。
- 浮窗正文统计行：`本周 ¥Y/150` —— 本周已得 / 本周上限；旁边彩条 = 阶段进度。
- 抽中若被上限截断，结果卡显示`本周已达上限，实发 ¥Z`（不骗人）。

## 到顶后用"非钱"继续激励（关键）
到达上限**不消耗抽奖券**（`econDrawBlockedReason()` 拦住，券留到下周期），并把抽奖按钮换成一条正向说明：
- **周顶**：`🎉 本周奖金已拿满 ¥150。抽奖券先留着，周一自动可用；这周继续做题只涨勋章和达标日，不亏。`
- **日顶**：`今天奖金已达上限 ¥40，日常券留到明天。现在做节点仍在攒达标日、离阶段大奖更近。`

到顶后仍在累积的"非钱"激励（都真实生效、不清零）：
1. **抽奖券留存**：到顶不扣券，下周期照抽（等于把大奖延后，不是没收）。
2. **达标日/连续/阶段**：继续攒，直接推进下一次阶段大券 + 连续加成。
3. **勋章（荣誉/收集层）**：P4 暑假勋章按节点数/达标日/整科完成发放，与预算无关、无上限，是到顶后的主要"继续做"的动力。
4. **进度可视**：能量环（今日）+ 阶段彩条（攒向下次大奖）持续动，给"看得见的前进感"。

设计意图：预算是"硬顶 + 到顶转非钱激励"，既不超支，又不会让学生在周中撞顶后"这周白学"——钱停了，荣誉和下期大奖的进度不停。
