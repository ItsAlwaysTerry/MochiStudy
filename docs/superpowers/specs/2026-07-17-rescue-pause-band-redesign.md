# 卡住救援“任务内暂停带”重设计

日期：2026-07-17

## 背景

现有“听不懂 / 视频太难”功能已经能给出精确书页、基础关键词、求助卡和延期回补，但救援面板被放进“学习材料与记录”抽屉，并同时展示原因、资源、勾选项、备注、检查题和多个按钮。它在视觉和交互上很像另一套正式任务，和正常的“看视频 → 做题 → 收尾 → 完成”混在一起。

救援的真实定位不是新任务，而是学生在当前任务中卡住时立刻处理的一次临时中断。本设计把它改为“任务内暂停带”，明确区分正常任务、临时救急和例外延期。

## 核心模型

页面只存在一条正常任务主线：

`看视频 → 做题 → 收尾 → 完成`

救援不新增步骤、不增加进度，也不进入正常任务计数。学生点击“听不懂 / 视频太难”后，当前任务临时暂停；救援结束后返回原步骤继续。

延期继续保留，但只作为救援区最末尾的例外出口。延期任务不算完成、不计奖励，并必须在本周复盘节点前回来处理。

## 界面结构

### 正常状态

- 正常四步进度条和当前步骤内容保持不变。
- “听不懂 / 视频太难”仍位于看视频区域附近，作为低干扰入口。
- “学习材料与记录”只包含例题截图、学习记录导入和本节收尾，不再包含救援内容。

### 临时救急状态

点击救援入口后，在四步进度条与正常步骤内容之间插入一条全宽暖色暂停带。

暂停带顶部固定显示：

`临时救急 · 不计任务进度`

暂停带出现时：

- 四步进度仍停留在原位置，例如“看视频”。
- 正常步骤内容降淡并显示“正常任务已暂停”。
- 正常任务主按钮暂时不可操作，避免学生同时操作两套流程。
- 任务标题和原视频信息继续可见，让学生知道正在处理哪个任务的卡点。

暂停带不是卡片列表、任务步骤或进度组件，不显示完成比例、奖励或待办数量。

## 暂停带内容

暂停带首屏只保留四部分：

1. 卡住原因：`概念没学过 / 公式太多 / 视频太长`。
2. “现在只做这一件事”：一个精确、有限的补救动作。
3. 返回检查：一句能否回主线的自检问题。
4. 主要出口：`解决了，回到视频` 和 `复制求助`。

删除以下容易造成“第二套任务”观感的内容：

- “已经翻过讲义”勾选项。
- “已经看过基础课”勾选项。
- 默认展开的“还卡在哪里”输入框。
- 同时平铺多个补救资源和后续题型。

点击“复制求助”后，才在暂停带内展开一个单行补充输入，让学生写清具体不懂之处，再复制求助卡。复制后暂停带继续保留，不能自动视为解决。

## 三个出口

### 解决了，回到视频

- 将 `support.status` 写为 `resolved`。
- 暂停带立即消失。
- 正常步骤内容和按钮恢复。
- 原任务进度、完成数和奖励均不改变。
- 页面保持原滚动位置。

### 复制求助

- 展开可选的具体卡点输入。
- 复制包含科目、原任务、卡住原因、精确救援范围和学生补充说明的求助卡。
- 状态继续保持 `active`，等待学生真正听懂后主动返回。

### 暂时放下

该入口只在暂停带最底部以弱化文字显示：

`实在无法继续？暂时放下，本周复盘前回来`

点击后：

- 将 `support.status` 写为 `deferred`。
- 允许同周其他未完成任务继续推进。
- 不增加完成数、不写入奖励完成时间、不计入日常奖励。
- 退出当前救援暂停带和当前任务队列。

## 延期提醒

普通页面不再显示大型“待补基础”任务区，也不把延期项渲染成正式任务卡。

有延期项时，只在计划顶部显示一条细提醒：

`有 1 个卡点暂时搁置 · 本周复盘前处理　查看`

点击“查看”才展开简短的例外列表，内容只有原任务名、卡住原因、回补期限和“回来处理”。

到物理第 7、14、21、28 天周复盘前，或数学/化学进入下一周前，未解决延期项继续触发现有拦截。拦截界面说明“这是之前暂时搁置的卡点”，不能伪装成新任务，也不能通过复制求助绕过。

## 万有引力内容

“万有引力”继续使用专门救援配置：

- 暂停《天体卫星所有题型》主线视频。
- 实体一轮讲义 2 第 27-30 页。
- 只补“万有引力定律、公式各字母含义、引力提供向心力”。
- 基础课关键词为“万有引力定律、向心力、卫星圆周运动”。
- 返回检查为“卫星为什么一直在下落，却没有直接掉到地面？”
- 暂时不展示同步卫星、双星、追及相遇和复杂比值题。

## 状态与兼容

继续使用现有计划状态中的 `tasks[taskId].support`：

```javascript
support: {
  status: "active" | "deferred" | "resolved",
  reason: "concept" | "formulas" | "long",
  note: string,
  openedAt: ISODateString,
  deferredAt: ISODateString,
  resolvedAt: ISODateString,
  helpCardCopiedAt: ISODateString
}
```

不新增 localStorage key，不改 `study_log` 和备份结构。旧状态中的 `attempted` 字段可以保留但不再在界面展示，避免数据迁移和旧版本恢复问题。

已有状态直接映射到新界面：

- `active`：显示任务内暂停带。
- `deferred`：显示顶部细提醒，并在回补节点参与拦截。
- `resolved` 或无 `support`：显示正常任务界面。
- 已真实完成的任务始终以完成状态为准，忽略旧救援状态。

## 交互与滚动

- 打开暂停带、切换原因、展开求助输入和关闭暂停带只局部更新当前任务，不重绘整个首页。
- 局部更新不能调用页面滚动或改变当前视口。
- “暂时放下”和“回来处理”需要重排任务队列时，刷新前释放按钮焦点，并锁定原始滚动坐标。
- 手机端暂停带采用单列布局，按钮触控高度不少于 40px，文字可以自然换行，不遮挡任务标题和四步进度。

## 异常处理

- 没有精确书页时，只显示已有概念和目录关键词，不编造页码。
- 外部资源失效时，书页和可复制关键词仍可用。
- 求助卡复制失败时，显示手动复制文本框，暂停带保持打开。
- `support` 字段损坏或值不合法时，按普通未完成任务处理，不能误判完成或发奖励。

## 验证标准

- 正常任务始终只有四步，救援不进入步骤条、完成计数或奖励进度。
- 救援面板不再出现在“学习材料与记录”抽屉。
- 点击救援入口后，正常任务明确显示暂停，且正常主按钮不可同时操作。
- 暂停带首屏不显示已尝试勾选项和备注输入框。
- 点击“复制求助”后才显示具体卡点输入，复制后仍保持救援状态。
- 点击“解决了”只恢复原任务，不直接完成任务。
- 点击“暂时放下”不计完成、不计奖励，同周可继续。
- 延期项平时只显示细提醒，到周节点仍会拦截。
- 万有引力只显示第 27-30 页、三个基础关键词和返回检查。
- 所有局部救援按钮点击后页面滚动位置不变。
- 旧 `active / deferred / resolved` 状态无需迁移即可正确显示。
- 执行 `node --check app.js` 和 `node --check modules/summerTasks.js`。
