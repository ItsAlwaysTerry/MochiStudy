# 题桌画布化重做设计（GoodNotes 式干净体验）

日期：2026-06-19
状态：已与用户确认方向，待审

## 1. 背景与动机

题桌 Phase 0–6 已全部落地（V5.x–V6.0）：粘图/上传/PDF → 套索框题 → 站内视觉 AI → 保存进学习档案 → 反向跳转，闭环都通。

但用户对题桌的**观感与交互**强烈不满：现在是「左中右三栏固定盒子」的布局——左侧文件栏、中间题图、右侧常驻 AI 面板，全是带边框和阴影的硬盒子，表单密集。用户原话：「很厚重、很老旧、那些操作逻辑太硬核、像东拼西凑拼起来」。

用户给了 GoodNotes 的 6 张参考截图，理想形态是「像一个很先进、很干净的笔记软件」：

- 文档铺满整屏，浅灰底 + 白纸，安静。
- 顶部只有一排纯图标细工具栏。
- 在文档上画虚线圈选一道题。
- 选完在题旁浮出小气泡问 AI。
- 气泡展开成可拖动浮窗（Chat + 笔记）。
- 关闭后题旁留一个小圆标；点小标重开橱窗（对话 + 总结的知识卡片）。
- 旧学习网站变成「另一个功能」，问过的题用一个按钮收集进学习档案。

**根因诊断**：GoodNotes 的高级感本质是「以文档为中心（canvas-first）」——文档占满屏，工具按需浮出；而现在的题桌是「以框为中心（chrome-first）」——三个永远撑着的盒子框住一张小图。这是观感老旧的根子。

## 2. 目标与非目标

### 目标

把题桌外壳从三栏固定盒子**彻底重做**成画布优先的浮层式体验：

- 文档铺满题桌视口（浅灰底、白纸、可滚动多页、可缩放拖动）。
- 顶部一排纯图标细工具栏。
- 左侧文件列表默认收起，点图标从左侧滑出抽屉，选完自动收起。
- 取消常驻右侧 AI 面板，改为「题旁小气泡 → 可拖动浮窗」。
- 关闭浮窗题旁留小圆标，点小标重开橱窗（对话 + 整理记录 + 收进学习档案）。
- 已收进档案的题，小标变金点，橱窗展示已存好的学习卡片。
- 视觉收口到克制的苹果式风格（浅灰/白纸/单一蓝、发丝描边、阴影只给浮层、平滑动效）。
- 移动端浮窗退化为底部弹层、抽屉全屏。

### 非目标（明确不做）

- **不做自由手写/画笔批注**（用户确认不需要；那是很重的绘图引擎）。
- 不改 `study_log` 单条字段结构。
- 不改备份文件 `version` / `data` 主结构。
- 不碰知识点预设列表、农场、勋章、复习、赛季系统。
- 不重写底层数据模型、套索几何、AI 调用、保存闭环——这些复用。
- 不引入后端依赖；保持纯前端 + localStorage + IndexedDB。

## 3. 用户体验设计

### 3.1 整屏布局

```
┌────────────────────────────────────────────────────────────┐
│ ☰  题桌            ⊕导入  ◌套索  ⊖75%⊕   🔍      ◔我的成长 │  纯图标细工具栏
├────────────────────────────────────────────────────────────┤
│                                                              │
│              ┌──────────────────────────┐                   │
│              │        白纸 · 卷子        │   浅灰底，文档铺满居中
│              │       （可滚动多页）       │   安静，文档是主角
│              │       （可缩放拖动）       │                   │
│              └──────────────────────────┘                   │
└────────────────────────────────────────────────────────────┘
```

- 顶部工具栏：纯图标 + 极小字，无大色块按钮框。
  - `☰`：开/关左侧文件抽屉。
  - `⊕导入`：粘贴/上传图片或 PDF（复用现有导入与 PDF.js 渲染）。
  - `◌套索`：进入/退出套索模式。
  - `⊖ % ⊕`：缩放控制。
  - `🔍`：搜索题图。
  - `◔我的成长`：进旧学习网站（`navigate("home")`）。
- 中间画布：浅灰背景、白纸居中。一张图 = 一页；PDF 多页竖向排列、可滚动；可缩放与拖动。无常驻右栏、无边框盒子。

### 3.2 文件抽屉

- 默认收起。点 `☰` 从左侧滑出，遮罩半透明，选中一张题图/一页 PDF 后自动收起。
- 内容沿用现有左栏信息：搜索、全部/未整理/数学/物理/化学、普通题图列表、PDF 可折叠分组（V5.11–V5.15 已有逻辑）。
- 抽屉是表现层重排，**不改**文件过滤、PDF 分组、重命名/删除等既有数据逻辑。

### 3.3 核心交互链：套索 → 气泡 → 浮窗 → 小标

1. 点 `◌套索` → 在卷子上画虚线圈住一道题，松手定形（复用 `questionDeskSelection.js` 几何 + 现有待确认框逻辑）。
2. 选区旁浮出**小气泡**：一个输入框「问 AI 关于这道题…」+ 发送键。轻、小、贴着选区。
3. 发送后气泡**展开成可拖动浮窗**（见 3.4）。
4. 浮窗可拖动、最小化、关闭。关闭后选区位置留一个**小圆标**（蓝点）。
5. 点小圆标 → 重新打开该题橱窗（即浮窗，见 3.4）。
6. 平时所有浮窗可收起，画布上只剩卷子 + 小圆标。

与现状最大差异：**取消常驻右侧 AI 面板**，AI 改为「哪道题要问就在哪道题旁浮出」。

### 3.4 浮窗 / 橱窗内容

同一个浮窗组件，既是「新问」也是「回看橱窗」。从上到下：

```
┌─────────────────────────────┐
│ 〔题2〕数学·导数  [缩略图] ✕ │  题号/科目/知识点 + 选区小截图
├─────────────────────────────┤
│  Chat                       │
│  你：这题第一步怎么想？        │
│  AI：先看…（多轮）            │  对话历史，可折叠
│  [ 继续问… ]          ↗发送   │
├─────────────────────────────┤
│  整理成学习记录               │
│  科目 [数学] 星级 ★★☆        │  清新卡片：科目 chip、星级点选
│  卡点：…                     │  关键字段（卡点/原题/套路）直接露出
│  原题：…                     │
│  今日套路：…                 │
│  ▸ 更多归档细节              │  错误类型/卡住步骤/关键突破/标签/信心分/耗时 默认折叠
│  ┌───────────────────────┐  │
│  │   ＋ 收进学习档案        │  │  一个按钮，存进旧学习网站
│  └───────────────────────┘  │
└─────────────────────────────┘
```

- 顶部：题号/科目/知识点（来自 `recognition`）+ 选区缩略图 + 关闭。
- Chat：复用现有 `chat` + `contextVersion`（调整框后「调整前旧对话」折叠逻辑保留）。
- 整理成学习记录：沿用 Phase 1.5 的清新卡片草稿表单（科目 chip、星级点星、关键字段露出、meta 折叠），呈现挪进浮窗。
- 「收进学习档案」按钮 = 现有保存闭环 `applyMochiRecord`。

### 3.5 与学习网站的连接

- 点「收进学习档案」→ 存进旧学习网站学习档案；农场成长、复习队列、勋章、今日报告照旧刷新；存成功 sparkle 庆祝（全部复用现有 `saveRecord` 逻辑）。
- 存完，题旁小圆标由蓝点变**实心金点**（已收藏态）；橱窗直接显示已存好的学习卡片（读 `study_log` by `savedLogId`）。
- 反向通路（V6.0 `openByLogId`）保留：学习档案卡片「看原题图」→ 跳回题桌并打开该题橱窗。
- 心智归位：题桌 = 当下圈题/问 AI/学懂；旧学习网站（首页/农场/勋章/复习/档案）= 平级「成长世界」，是题桌之外的另一个功能，问过的题汇进去成为学习资产。

## 4. 视觉系统

- 配色 token：画布浅灰 `#f4f6f9`、纸面纯白 `#ffffff`、文字近黑 `#1c1c1e`、次要灰 `#64748b`、**单一强调蓝** `#2563eb`、发丝描边 `#e6eaf0`。
- 去掉常驻 chrome 的粗边框、色块和重阴影；常驻元素用发丝描边。
- **阴影只给浮层**（浮窗、气泡、抽屉、小标）——浮起来的东西才有柔和投影，层次清楚又安静。
- 工具栏：纯图标按钮，hover 才出淡底，无填充色块大按钮。
- 圆角：浮窗/气泡 14–16px；卡片 12px。
- 动效：气泡→浮窗弹性展开、抽屉滑出、小标弹入、浮窗拖动跟手——平滑过渡是高级感的重要来源。
- 字号克制：主字段清楚，次要信息灰且小。

## 5. 技术架构

### 5.1 复用（不重造）

- 数据模型：`question_desk_images / question_desk_items / question_desk_ui_state` + IndexedDB `question_images_blob`。item 字段 `rect / chat / recognition / recordDraft / savedLogId / contextVersion` 全部沿用。
- 套索几何：`questionDeskSelection.js`（lasso/rect 纯函数、坐标换算、marker 定位）。
- AI 提示与识别：`questionDeskAI.js`。
- 保存闭环：`window.MochiApp.applyMochiRecord` + `study_card_meta`。
- 反向跳转：V6.0 `window.MochiQuestionDesk.openByLogId / hasImageForLog`。
- 路由外壳：`body.desk-mode`、`#desk` 顶层路由、`navigate("home")/("desk")`。

### 5.2 重写（表现层）

- `modules/questionDesk.js` 的 `render` 及其 `renderSidebar / renderViewer / renderPanel` 三栏渲染 → 画布优先渲染：顶部图标工具栏 + 全幅滚动/缩放画布 + 滑出抽屉。
- 取消常驻右侧 `qd-panel`，新增浮层组件：题旁小气泡 + 可拖动浮窗（Chat + 草稿 + 收进档案）。
- `style.css` 的 `.question-desk / .qd-sidebar / .qd-canvas / .qd-panel` 等块按新视觉系统重写；新增浮层、抽屉、工具栏、小标样式。
- 因 `questionDesk.js` 已 3000+ 行，浮窗/橱窗组件**拆为单独模块**（暂名 `modules/questionDeskFloat.js`），`questionDesk.js` 保留外壳渲染、事件编排、存储调用。新模块经 `index.html` 加 `?v=` 引入。

### 5.3 坐标与缩放

- 画布缩放/滚动时，气泡、浮窗锚点、小标位置都基于选区 `rect` 的相对坐标，经统一换算到当前画布的像素位置（复用/扩展 `questionDeskSelection.js` 的换算），避免缩放后错位。
- 浮窗可被用户拖到任意位置（拖动后记内存态位置，不必持久化）；小标始终锚在 `rect`。

### 5.4 红线约束

- 不改 `study_log` 字段、备份 `version/data` 结构、知识点预设列表、农场/勋章/复习系统。
- 每改完一个 JS 文件 `node --check`。
- 图片不写 localStorage（仍走 IndexedDB）。

## 6. 实现分步（每步可单独跑、单独验）

1. **换外壳**：三栏 grid → 画布优先布局 + 顶部图标工具栏 + 左侧滑出文件抽屉；套索/查看器照常工作在新画布里。`node --check` + 手验导入/选图/PDF 仍正常。
2. **浮层化 AI**：删常驻右面板，做「题旁小气泡 → 可拖动浮窗（Chat + 整理记录）」，接现有 AI 与保存。手验问答、生成草稿、保存。
3. **小标 + 橱窗**：关闭留小标、点小标重开橱窗、已收藏小标变金点、收进档案打磨。手验回看与反向跳转。
4. **视觉与动效统一 + 移动端**：配色/间距/动效收口；手机浮窗退化为底部弹层、抽屉全屏。

每步结束：`node --check` 全部题桌模块、`index.html` 版本号递增、手动验收、必要时更新 CLAUDE.md。

## 7. 风险与缓解

- **浮窗/小标在缩放滚动时定位**：复用套索坐标换算；浮窗拖动用内存态。
- **大改渲染层碰到保存闭环回归**：数据与保存函数原样不动，只换表现层；分步验收。
- **`questionDesk.js` 体量**：浮窗拆独立模块，避免单文件继续膨胀。
- **移动端空间**：浮窗 → 底部弹层；抽屉全屏；套索在触屏的手感单独验。
- **PDF 多页画布滚动性能**：沿用现有逐页 Blob 渲染，按需 hydrate，避免一次性全渲染。

## 8. 验收标准

- 打开题桌：文档铺满、浅灰底白纸、顶部纯图标工具栏、无常驻右栏。
- 文件列表默认收起，点 `☰` 滑出、选完自动收起。
- 套索圈题 → 题旁浮气泡 → 展开可拖动浮窗 → 问答多轮 → 生成草稿 → 收进学习档案，全程在题桌内完成。
- 关浮窗题旁留小标；点小标重开橱窗；已收藏小标为金点并显示已存学习卡片。
- 学习档案卡片「看原题图」仍能跳回题桌打开对应橱窗。
- 整体观感克制干净（浅灰/白纸/单一蓝、阴影只给浮层、动效平滑），不再有「厚重、拼凑」感。
- 移动端可用：抽屉全屏、浮窗底部弹层。
- `study_log` / 备份结构 / 知识点预设 / 农场勋章复习 均未改动；所有 JS `node --check` 通过。
