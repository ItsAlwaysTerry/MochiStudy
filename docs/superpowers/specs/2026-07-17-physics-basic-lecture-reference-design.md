# 全站物理任务基础讲义精确定位设计

日期：2026-07-17

## 背景

物理任务的“看不懂 / 视频太难”救急区目前主要显示一轮复习视频和一轮讲义位置，例如：

`黄夫人物理一轮 · 视频 P1 · 讲义 33 · 牛二分析运动`

学生手中还持有三册更基础的《黄夫人高一高二讲义》。对基础薄弱学生而言，救急时往往应先翻基础讲义，再决定是否进入一轮复习内容。只显示一轮讲义会遗漏更合适的入口。

本设计覆盖网站当前全部物理任务，不只修改“牛顿第二定律”或某一天。覆盖范围包括首页的详细物理任务、28 天路线中由视频条目生成的任务卡、周测回炉任务和专题任务。

## 已确认资料

基础讲义来源为：

- `73面 讲义一.pdf`：平衡力学、牛顿力学。
- `83面 讲义三.pdf`：电场、恒定电流、磁场、电磁感应。
- `92面 讲义二.pdf`：曲线运动、天体运动、功能、动量。

网站现有一轮讲义来源为：

- `【一轮复习讲义】1.运动学-力学-牛二.pdf`
- `【一轮复习讲义】2.曲线-天体-功能-动量.pdf`
- `【一轮复习讲义】3.电场电-学实验-磁场-电磁感应.pdf`

所有学生端页码以讲义页面上印刷的实体书页码为准。PDF 页序只用于开发核对，不展示给学生。

## 方案选择

采用逐任务人工核对的精确映射表，不使用关键词运行时自动匹配。

未采用的方案：

- 按大专题统一给出整章范围：维护简单，但学生仍需在几十页中自行寻找，不能解决“看不懂时不知道翻哪里”。
- 根据任务标题自动匹配关键词：扩展方便，但“牛二”“正交分解”“电源实验”等交叉主题容易匹配到错误章节。

逐任务映射的维护量更高，但页码可靠、界面可预测，符合本功能的目的。

## 映射范围

每一张可进入救急状态的物理任务卡都必须有明确结果：

1. 有基础讲义对应内容：保存册号、章节、小节和实体书页码。
2. 基础讲义没有覆盖：明确标记使用哪个现有基础视频或学校资料，不编造章节和页码。

映射对象包括：

- `PHYSICS_TASKS` 中的全部详细任务。
- `PHYSICS_ROUTE_VIDEO_LIBRARY` 中实际被 28 天路线使用的全部视频条目。
- 只有 `mission`、没有指定视频的路线任务。
- 周测、错题回炉和公式回炉中可单独打开的具体任务。

同一任务涉及多个知识点时，只保留最直接的 1 至 2 个基础小节。优先级为：先决概念、当前视频核心内容、最基础例题。进阶模型不塞入首次救急范围。

## 数据结构

新增集中式只读映射 `PHYSICS_BASIC_LECTURE_MAP`，键使用详细任务 id 或路线视频 key，避免在渲染时根据中文标题猜测。

```javascript
const PHYSICS_BASIC_LECTURE_MAP = {
  [taskIdOrVideoKey]: {
    book: String,
    chapter: String,
    section: String,
    pages: String
  },
  "vibration-basic": {
    unavailable: true,
    fallbackLabel: "机械振动基础视频"
  }
};
```

正式映射必须在核对 PDF 后填写真实章节、小节和实体页码，代码中不得保留占位值。

路线任务生成时保存其来源视频 key，救急渲染通过该 key 查表。详细任务直接通过 task id 查表。没有映射记录的任务在开发验证中视为错误，不能静默隐藏。

不新增 localStorage key，不修改 `summer_task_state`、`study_log` 或备份格式。映射是静态内容数据，不属于学生进度。

## 救急区展示

救急区的资源顺序固定为：

1. 基础讲义对应位置。
2. 一轮复习对应位置。
3. 必要时显示基础视频或目录关键词。

有基础讲义映射时显示：

```text
基础讲义对应位置
黄夫人高一高二讲义{册号} · {章节} · {小节标题} · 实体书{页码范围}
```

一轮位置继续显示：

```text
一轮复习对应位置
黄夫人物理一轮 · 视频 P1 · 讲义33 · 牛二分析运动
```

若基础讲义未收录该专题，则第一行改为：

```text
基础讲义未收录
本专题先使用「机械振动基础视频」
```

基础讲义和一轮讲义使用两个独立的资源行，不合并成一段长文字。基础讲义行使用书本图标，一轮复习行保留视频图标。

## 去重规则

当前救急数据中的 `prep.oneRound`、`prep.rescue.book` 和精确的一轮视频位置可能同时指向一轮资料。改造后必须避免重复展示：

- 有精确一轮视频位置时，显示“视频 P + 一轮讲义页码 + 标题”。
- 基础讲义使用新的独立映射，不复用 `prep.rescue.book`。
- `prep.oneRound` 仅在没有精确一轮视频定位时作为一轮讲义范围兜底。
- 基础视频关键词只在需要补充或基础讲义未收录时显示。

## 内容核对规则

- 逐页核对三册基础讲义的章节起止位置和页面印刷页码。
- 章节、小节标题按讲义原文记录，不自行改写成网站任务标题。
- 一个小节跨页时记录完整连续范围。
- 同一页同时出现两个小节时，可按实际相关内容共用该页。
- PDF 封面、空白页、目录页不计入实体书页码。
- 映射不确定时标记为“基础讲义未确认”，继续核对后再上线，不能凭经验猜页码。

## 异常与兼容

- 旧任务状态无需迁移，刷新后直接使用新静态映射。
- 基础讲义未收录的专题仍可使用现有基础视频，不影响救急流程。
- 一轮视频缺少精确 P 号时，保留现有一轮讲义范围，不伪造 P 号。
- 路线中新增物理任务但没有新增映射时，开发验证应列出缺失 key。
- 已经进入 `active` 或 `deferred` 救急状态的任务，刷新后立即显示新的基础讲义位置。

## 验证

- 枚举全部物理详细任务和 28 天路线任务，确认每个任务都有基础讲义映射或明确的未收录兜底。
- 抽查讲义一的平衡力学、牛顿力学任务，章节、小节和实体页码与 PDF 页面印刷页码一致。
- 抽查讲义二的曲线运动、天体运动、功能任务。
- 抽查讲义三的电场、恒定电流、磁场、电磁感应和电学实验任务。
- “牛二分析运动”救急区同时显示基础讲义和一轮复习两行。
- 万有引力等自定义救急任务不再重复显示两条一轮讲义信息。
- 振动、热学等未收录专题显示明确的基础视频或学校资料兜底，不显示虚假页码。
- 移动端长章节名正常换行，不挤压按钮或造成横向滚动。
- 打开、切换原因和关闭救急区不改变页面滚动位置。
- 执行 `node --check modules/summerTasks.js` 和 `node --check app.js`。
