## 五角色分析结论（已完成）

### 诺曼发现
- 首页「今日复习」卡显示 `还记得吗？` + 立即展示卡点 → 被动展示答案，和主动回忆的设计意图矛盾
- 复习页「今日建议」和「待处理」同一个条目出现两次 → 用户不知道要做哪个
- 赛季徽章（`.season-badge`）不可点击，用户看到倒计时但无法深入 → 死路

### 奥克利发现
- 首页复习卡的卡点直接可见：学生看到即产生「认知幻觉」（觉得自己记得），不需要主动提取 → 最大学习漏洞
- 「今日建议」同时展示主要卡点 + 为什么今天 → 两列信息叠加，决策时间过长，基础差的学生容易放弃

### 拉姆斯发现
- `renderDailyGoalDots()` 函数（farm.js L224–243）仍存在于代码中，但 V2.0 已删除所有调用点，是纯死代码 → 视觉负担无，但维护负担存在
- 复习页「今日建议」的 `.review-task-grid` 两列（主要卡点 + 为什么今天）信息密度过高，超出用户的注意力容量

### 福勒发现
- `renderDailyGoalDots` 是无调用的孤立函数，应删除
- `filterItems()` 没有过滤掉已在 `todaySuggestions` 里的条目，导致复习页两个区域出现重复项

### 芒格裁决（打分矩阵）

### 改动 1：首页复习卡卡点 → `<details>` 主动回忆翻转

**发现来源**：诺曼 + 奥克利联合  
**问题**：`renderTodayReviewCard()` 在首页直接展示 `mainPainPoint`，学生看到就产生「认知幻觉」，不需要大脑主动提取。「还记得吗？」的文案配上立即可见的答案，完全自相矛盾。  
**方案**：用原生 `<details>/<summary>` 包裹卡点，默认收起，学生主动点击才展开，零 JavaScript，零风险。

**文件**：`modules/farm.js`  
**函数**：`renderTodayReviewCard()`  
**定位**：找到以下代码（约第 275 行）：

```javascript
${item.mainPainPoint ? `<p class="home-review-pain"><span class="home-review-recall-hint">还记得吗？</span>${escapeAttr(item.mainPainPoint)}</p>` : ""}
```

**替换为**：

```javascript
${item.mainPainPoint ? `
<details class="home-review-spoiler">
  <summary class="home-review-recall-hint">还记得吗？先想一想，再展开看卡点</summary>
  <p class="home-review-pain">${escapeAttr(item.mainPainPoint)}</p>
</details>
` : ""}
```

**CSS 新增**（`style.css`，追加到 `.home-review-pain` 规则块附近，约第 1361 行之后）：

```css
.home-review-spoiler {
  margin: 4px 0 8px;
  cursor: pointer;
}
.home-review-spoiler summary {
  font-size: 13px;
  list-style: none;
  color: var(--text-muted);
  user-select: none;
}
.home-review-spoiler summary::-webkit-details-marker { display: none; }
.home-review-spoiler[open] summary {
  color: var(--text-secondary);
  margin-bottom: 4px;
}
```

**验证**：
1. `node --check modules/farm.js` → 无报错
2. 浏览器打开 `index.html`，确保有复习建议（有历史导入记录）
3. 首页「今日复习」卡片，卡点区域显示「还记得吗？先想一想，再展开看卡点」而非卡点文字
4. 点击该行，展开后显示卡点文字
5. 再次点击，收起

**预期效果**：每天打开 App 看到复习卡时，必须主动点击才能看到答案，强制主动回忆。

### 改动 2：赛季徽章变为可点击按钮（跳转到赛季页）

**发现来源**：诺曼  
**问题**：`.season-badge` 是纯展示 `<div>`，显示赛季名和倒计时，但不可交互。用户看到「还剩 12 天」想了解更多，找不到入口。  
**方案**：改为 `<button data-route="season">`，app.js 的全局导航点击代理已处理所有 `data-route`，无需额外 JS。

**文件**：`modules/farm.js`  
**函数**：`renderFarm()` 内的 `seasonBanner` IIFE  
**定位**：找到以下代码（约第 386 行）：

```javascript
return `<div class="season-badge${isEndingSoon ? " ending-soon" : ""}">${icon} ${escapeAttr(currentSeason.name)} · ${countdown}</div>`;
```

**替换为**：

```javascript
return `<button class="season-badge${isEndingSoon ? " ending-soon" : ""}" data-route="season" type="button">${icon} ${escapeAttr(currentSeason.name)} · ${countdown}</button>`;
```

**CSS 修改**（`style.css`，找到 `.season-badge` 规则块，约第 3019 行）：  
在现有规则末尾追加以下属性，让 `<button>` 重置浏览器默认样式：

```css
.season-badge {
  /* 已有属性保持不变，在末尾追加： */
  border: none;
  cursor: pointer;
  font-family: inherit;
  font-size: inherit;
  font-weight: inherit;
  line-height: inherit;
  text-align: left;
}
```

> ⚠️ 注意：不要覆盖已有属性，只在现有 `.season-badge {}` 规则块内追加这 7 行。

**验证**：
1. `node --check modules/farm.js` → 无报错
2. 浏览器打开，首页顶部赛季徽章可点击（鼠标悬停显示手形光标）
3. 点击后跳转到赛季页（`#season`）
4. 赛季页正常渲染，点击浏览器后退 / 底部导航首页，回到首页正常

### 改动 3：复习页「待处理」去重（排除今日建议里的条目）

**发现来源**：诺曼 + 福勒  
**问题**：`filterItems()` 只过滤了 status 为 `stable`/`consolidating` 的条目，没有排除已在 `todaySuggestions` 里的条目。用户在「今日建议」看到「数学 · 导数」，往下滚动「待处理」里还有同一条。  
**方案**：`filterItems` 增加第二个参数 `todaySuggestions`，用 `Set` 快速过滤。

**文件**：`modules/reviewPage.js`

**修改一**：找到 `filterItems` 函数（约第 218 行）：

```javascript
function filterItems(items) {
    return items.filter((item) => {
      if (STATE.subjectFilter !== "all" && item.subject !== STATE.subjectFilter) return false;
      return !["stable", "consolidating"].includes(item.status) && item.score > 0;
    });
  }
```

**替换为**：

```javascript
function filterItems(items, todaySuggestions) {
    const skipKeys = new Set((todaySuggestions || []).map((s) => s.key));
    return items.filter((item) => {
      if (skipKeys.has(item.key)) return false;
      if (STATE.subjectFilter !== "all" && item.subject !== STATE.subjectFilter) return false;
      return !["stable", "consolidating"].includes(item.status) && item.score > 0;
    });
  }
```

**修改二**：在 `render()` 函数里找到调用 `filterItems` 的那行（约第 18 行）：

```javascript
const filteredItems = filterItems(reviewState.items);
```

**替换为**：

```javascript
const filteredItems = filterItems(reviewState.items, reviewState.todaySuggestions);
```

**验证**：
1. `node --check modules/reviewPage.js` → 无报错
2. 浏览器打开复习页（有历史数据）
3. 若「今日建议」里有「数学 · 导数」，「待处理」列表里不应再出现同一条
4. 筛选按钮（数学/物理/化学/全部）切换后，今日建议里的条目仍然不出现在待处理里
5. 若「今日建议」为空，「待处理」显示所有待复习条目（不丢失）

### 改动 4：复习页「今日建议」删掉「为什么今天」列

**发现来源**：拉姆斯 + 奥克利  
**问题**：`renderTodayTask()` 里的 `.review-task-grid` 有两列：「主要卡点」和「为什么今天」。对于一个基础差、容易受挫的学生，决策时信息越少越好。「为什么今天」的内容（`primaryReason`）已经在「待处理」列的副文字里显示，今日建议卡里不需要重复。  
**方案**：删掉整个 `.review-task-grid`，只保留 `mainPainPoint` 作为单行文字。

**文件**：`modules/reviewPage.js`  
**函数**：`renderTodayTask()`  
**定位**：找到以下代码（约第 273–283 行）：

```javascript
          <div class="review-task-grid">
            <div>
              <small>主要卡点</small>
              <p>${escapeHtml(item.mainPainPoint || "没有明确卡点，适合轻量回顾。")}</p>
            </div>
            <div>
              <small>为什么今天</small>
              <p>${escapeHtml(item.primaryReason)}</p>
            </div>
          </div>
```

**替换为**：

```javascript
          <p class="review-task-pain">${escapeHtml(item.mainPainPoint || "没有明确卡点，适合轻量回顾。")}</p>
```

**CSS 新增**（`style.css`，追加到 `.review-task-grid` 规则附近，约第 4406 行之后）：

```css
.review-task-pain {
  margin: 4px 0 8px;
  font-size: 15px;
  color: var(--text);
  line-height: 1.5;
}
```

**验证**：
1. `node --check modules/reviewPage.js` → 无报错
2. 浏览器打开复习页，「今日建议」卡片只显示：科目 · 知识点 · 状态 + 卡点文字 + 两个按钮
3. 不再有「主要卡点」/ 「为什么今天」两个带 `<small>` 标题的列
4. 卡片高度应略有缩减（视觉更紧凑）

### 改动 5：删除 `renderDailyGoalDots` 死代码

**发现来源**：福勒  
**问题**：`renderDailyGoalDots` 在 V2.0 删除 `daily-goal-compact` 区块时失去了所有调用点，成为孤立函数。用 grep 确认：全项目只有 `modules/farm.js:224` 一处定义，无任何调用。  
**方案**：直接删除整个函数体。

**执行前验证**（先跑这条命令，确认 0 调用才执行删除）：
```
grep -rn "renderDailyGoalDots" .
```
预期结果：只有 `modules/farm.js:224:  function renderDailyGoalDots()` 这一行。若有其他调用，**不要执行本改动，报告给用户**。

**文件**：`modules/farm.js`  
**定位**：找到以下函数（约第 224–243 行），整段删除：

```javascript
  function renderDailyGoalDots() {
    const state = window.MochiApp?.getDailyTaskState?.() || {};
    const today = new Date().toISOString().slice(0, 10);
    const todayLogs = (window.MochiApp?.readStudyLogs?.() || []).filter((log) => String(log.date || "").slice(0, 10) === today);
    const labels = { math: "数学", physics: "物理", chemistry: "化学" };
    return SUBJECTS.map((subject) => {
      const done = state[subject]?.completed;
      const todayLog = done ? todayLogs.find((log) => log.subject === subject) : null;
      const nodeLabel = todayLog?.nodeLabel || "";
      return `
        <span class="goal-dot ${done ? "done" : ""}">
          <span class="material-symbols-outlined">${done ? "check_circle" : "radio_button_unchecked"}</span>
          <span class="goal-dot-label">
            <strong>${labels[subject] || subject}</strong>
            ${nodeLabel ? `<em>${escapeAttr(nodeLabel)}</em>` : ""}
          </span>
        </span>
      `;
    }).join("");
  }
```

**替换为**：（空，整段删除）

**验证**：
1. `node --check modules/farm.js` → 无报错
2. 浏览器刷新首页，功能正常（农场、导入、streak 横幅全部正常显示）
3. grep 确认 `renderDailyGoalDots` 不再存在于任何文件中
