可以，我们把它拆成一轮很稳的改动。我的建议是：**先不大改架构，不碰 `study_log`，不改复习算法，只改“学生每天真实会怎么走”的表面路径。**

**我建议的改动顺序**
第一优先级：**首页加“今日复习”卡片**

这是最关键的一刀。因为用户画像是“不主动复习”，所以不能指望她每天主动点进复习页。

首页右侧现在已经有今日目标、农场、导入入口。我们可以加一个很轻的卡：

```text
今日复习
函数
原因：1 次低星，10 天没碰，主要卡在定义域限制
[开始复习] [看全部]
```

交互我建议这样：

- 首页只显示 `todaySuggestions[0]`，最多一个。
- 点“开始复习”：
  - 自动复制该知识点复习包。
  - 跳到复习页。
  - 自动展开对应知识点的回填入口。
- 这样首页不增加第二套回填逻辑，避免复制 `reviewPage` 的功能。
- 学生感受是“一点就开始”，代码上仍然保持复习闭环集中在 `reviewPage`。

这比在首页也做一套粘贴导入更干净。

第二优先级：**学习档案加“整理模式”**

现在学习档案有编辑、删除、拖拽、摘要编辑，这些功能都合理，但默认全露出来会像后台。

我建议默认状态：

- 显示知识点行。
- 展开后先显示核心摘要。
- 历史卡片折叠。
- 卡片默认只保留“看套路 / 看原题”这类学习动作。
- 隐藏删除、拖拽、编辑按钮。

然后加一个小按钮：

```text
[整理]
```

打开整理模式后才出现：

- 编辑卡片
- 删除卡片
- 拖拽排序
- 编辑核心摘要

这里有一个设计取舍：**摘要编辑要不要也藏进整理模式？**  
我倾向于藏。因为“摘要校正”虽然重要，但它是整理动作，不是每天学习动作。导入复习结果后可以单独提醒更新摘要。

第三优先级：**复习导入成功后提示“沉淀一句精华”**

这是让学习档案真正变薄的关键。

导入复习结果成功后，不弹大弹窗，只在当前复习项下面出现一句：

```text
这次有新的突破吗？
[写入核心摘要]
```

点了以后跳到学习档案对应知识点，并自动打开摘要编辑，或者在复习页内弹一个极简三字段编辑区。

我更推荐第一版做简单一点：

- 导入成功后显示按钮“去更新核心摘要”。
- 点击后跳学习档案，定位知识点，打开摘要编辑。
- 不在复习页再造一套摘要编辑表单。

这样职责边界最清楚。

第四优先级：**复习页继续减压**

复习页现在结构没问题，但“全部待复习”容易变成列表压力。

建议改成：

```text
今日建议
全部待复习
已冷却/近期稳定
最近复习
```

默认“全部待复习”只放真正需要处理的：

- 待复习
- 仍需继续
- 到期的待巩固

把“近期稳定”和“未到期待巩固”默认放到折叠区，文案不要叫“全部”，可以叫：

```text
待处理
```

这很重要。学生不需要看到“我还有一大堆稳定的东西”。

第五优先级：**文档和导航收口**

文档上补：

- `study_node_summary` 正式进入 localStorage 结构说明。
- 首页是主入口。
- 复习页是完整指挥台。
- 学习档案默认不是管理页，整理能力放到整理模式。

导航上我建议暂时不大动桌面端，但移动端可以更坚定：

```text
首页 / 复习 / 档案 / 日历 / 设置
```

赛季、勋章继续存在，但作为首页反馈入口或侧边栏入口，不放移动底部主导航。

**我建议本轮实际执行范围**
不要一次把所有东西都改爆。最稳的一轮是：

1. 首页新增“今日复习”卡片。
2. 点“开始复习”复制复习包并跳转复习页展开对应项。
3. 学习档案加“整理模式”，默认隐藏编辑/删除/拖拽/摘要编辑。
4. 复习导入成功后显示“去更新核心摘要”。
5. README / AGENTS / review-module 文档同步。

这轮做完，整个产品气质会明显变成：

```text
首页负责今天做什么。
复习页负责完整复习闭环。
档案负责沉淀精华。
管理能力退到整理模式里。
```

我建议就按这个顺序动手。第一刀从首页“今日复习”开始，因为它最直接改善真实使用工作流。
