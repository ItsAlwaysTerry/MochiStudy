# 第四轮：整理模式入口视觉提示（已完成）

## 问题

学习档案顶部"整理"按钮：
- 新用户不知道编辑/删除/拖拽功能藏在整理模式里
- 整理模式开启后页面没有明显视觉区分
- 按钮缺少功能说明，初次使用无法理解

## 改动（modules/knowledgeMap.js + style.css）

1. 整理按钮加 `title` tooltip：
   - 未开启："开启后可编辑、删除和拖拽卡片"
   - 已开启："退出整理模式"
2. 整理模式激活时，卡片列表上方插入状态条（`.organize-mode-bar`）：
   "整理模式 · 可编辑、删除和拖拽卡片" + 内嵌"完成整理"按钮
3. 状态条为淡紫色背景 + 紫色边框，和整理按钮颜色呼应
4. 状态条内的"完成整理"使用 `data-card-organize`，和顶部按钮同一个处理器

## 设计原则

管理能力隐藏在整理模式里，但入口要足够明确，不能让用户以为功能不存在。
状态条是"你现在处于特殊模式"的环境提示，颜色柔和不抢眼。
